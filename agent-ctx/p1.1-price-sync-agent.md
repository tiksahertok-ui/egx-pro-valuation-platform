# P1.1: Real-Time EGX Data Pipeline - Price Sync Worker

## Task Summary
Created the price-sync cron worker at `mini-services/price-sync/` for the EGX Pro Valuation Platform. This standalone service fetches EGX stock prices from Yahoo Finance, falls back to hardcoded financial data when APIs are unavailable, and updates the Supabase PostgreSQL database on 15-minute intervals during EGX trading hours.

## Files Created

### 1. `mini-services/price-sync/package.json`
- Standalone Bun project with `@supabase/supabase-js` dependency
- Scripts: `sync` (one-shot), `daemon`/`dev` (continuous with hot reload)

### 2. `mini-services/price-sync/tsconfig.json`

### 3. `mini-services/price-sync/sync.ts` - Main sync worker (~1050 lines)
Complete, production-ready sync worker with:

**Data Sources:**
- Primary: Yahoo Finance API (`query1.finance.yahoo.com/v8/finance/chart/{TICKER}.CA`)
- Fallback: Full hardcoded financial data for 95+ EGX stocks (from `egx-financial-data.ts`)

**Database Operations:**
- Reads stocks from Supabase `Stock` table
- Updates `Stock` table with price, marketCap, PE, PB, EPS, bookValuePerShare, 52-week high/low, etc.
- Inserts/updates `PriceHistory` records with OHLCV data (adapts to actual Supabase schema with `source` column)
- Inserts/updates `FinancialData` records (maps hardcoded data to actual Supabase schema with `quarter`, `costOfRevenue`, `grossProfit`, etc.)
- Logs sync activity to `DataRefreshLog` table (gracefully handles missing table)

**Market Hours:**
- EGX trading: Sunday-Thursday, 10:00-14:30 Cairo time (UTC+2, no DST)
- Only syncs during trading hours unless `--force` flag is used
- Daemon mode calculates time until next market open and sleeps accordingly

**Error Handling:**
- Retry logic with exponential backoff (3 retries, 2s base delay)
- Rate limiting (250ms between Yahoo Finance API requests)
- 12-second timeout per Yahoo request
- RLS permission detection at startup with clear warnings
- Graceful handling of missing tables, blocked writes, and API failures

**Modes:**
- One-shot: `bun run sync.ts --once [--force]`
- Daemon: `bun run sync.ts` (runs continuously with 15-min intervals)

## Test Results
- Supabase connection: OK (reads work with anon key)
- Yahoo Finance: Successfully fetched live prices for 18/21 stocks (e.g., COMI at 81.20, EAST at 21.80, ETEL at 32.77)
- Market hours check: Correctly skips sync when market is closed (without --force)
- RLS detection: Properly detects and warns about write permission issues at startup
- Error handling: All errors caught and logged gracefully
- 2 stocks (BNQA, MHMD) not found on Yahoo Finance (correct - these aren't standard EGX tickers)

## Known Issues
- Database writes are currently blocked by RLS because the `.env` has `SUPABASE_SERVICE_ROLE_KEY` set to the anon key (`sb_publishable_...`) instead of the real service role key (JWT starting with `eyJ...`). The sync worker detects this at startup and provides clear guidance.
- The `DataRefreshLog` table doesn't exist in the deployed Supabase schema. The sync worker handles this gracefully with a warning.
- Some EGX tickers in the database (BNQA, MHMD) don't have Yahoo Finance equivalents.

## Architecture Notes
- The sync worker is intentionally standalone — it does NOT import from the main Next.js project
- All hardcoded financial data is self-contained within `sync.ts` to ensure independent operation
- The Yahoo Finance symbol is constructed as `{TICKER}.CA` since the `yahooSymbol` column may not exist in the deployed Supabase schema
- PriceHistory and FinancialData insert operations adapt to the actual Supabase table schemas (which differ slightly from the Prisma schema)
